# Qiskit Python Export

Export KetGrid circuits to executable Qiskit Python code.

## Overview

KetGrid can export quantum circuits to Python code compatible with [Qiskit](https://qiskit.org/)'s `QuantumCircuit` API. This allows users to:
- Run circuits on IBM quantum hardware
- Use Qiskit's simulators and tools
- Share circuits with the broader quantum computing community
- Integrate with Qiskit ecosystem libraries

## Usage

### GUI Export

1. Open or create a circuit in KetGrid
2. Select **File → Export to Qiskit…**
3. Choose a save location (`.py` extension recommended)
4. The generated Python file can be run with Qiskit

### Programmatic Export

```rust
use ketgrid_core::Circuit;
use ketgrid_core::format::qiskit::circuit_to_qiskit;

let circuit = Circuit::new(2);
// ... add gates ...

// Export to Python code
let python_code = circuit_to_qiskit(&circuit)?;
std::fs::write("bell_state.py", python_code)?;
```

Or use the convenience method:

```rust
let python_code = circuit.to_qiskit()?;
```

## Gate Mapping

| KetGrid Gate | Qiskit Code | Notes |
|--------------|-------------|-------|
| `H` | `qc.h(q)` | Hadamard gate |
| `X` | `qc.x(q)` | Pauli-X (NOT) |
| `Y` | `qc.y(q)` | Pauli-Y |
| `Z` | `qc.z(q)` | Pauli-Z |
| `S` | `qc.s(q)` | Phase gate |
| `T` | `qc.t(q)` | T gate |
| `Rx(θ)` | `qc.rx(θ, q)` | X rotation |
| `Ry(θ)` | `qc.ry(θ, q)` | Y rotation |
| `Rz(θ)` | `qc.rz(θ, q)` | Z rotation |
| `Cnot` | `qc.cx(ctrl, target)` | Controlled-NOT |
| `Cz` | `qc.cz(ctrl, target)` | Controlled-Z |
| `Swap` | `qc.swap(q1, q2)` | Swap two qubits |
| `Toffoli` | `qc.ccx(c1, c2, t)` | CCNOT (two controls) |
| `Barrier` | `qc.barrier([q])` | Visual separator |
| `Identity` | (skipped) | No-op, omitted from output |
| `Custom(name)` | `# Custom gate: name` | Comment only |

## Output Format

Generated Python code follows this structure:

```python
# Generated by KetGrid - Qiskit QuantumCircuit export

from qiskit import QuantumCircuit

qc = QuantumCircuit(2, 2)  # 2 qubits, 2 classical bits

qc.h(0)
qc.cx(0, 1)

# Measurements
qc.measure(0, 0)  # Measure qubit 0 to classical bit 0
qc.measure(1, 1)  # Measure qubit 1 to classical bit 1

# Run the circuit
# from qiskit import transpile, Aer
# simulator = Aer.get_backend('aer_simulator')
# ...
```

## Execution Order

Gates are sorted by their column position before export, ensuring the generated code reflects the correct circuit execution order. This matches how KetGrid simulates circuits left-to-right.

## Measurements

- Each measurement operation generates a `qc.measure(q, c)` call
- Classical bits are allocated automatically based on measurement count
- Measurements are numbered sequentially (0, 1, 2, …) in column-sorted order

## Error Handling

The export can fail with:

- `QiskitError::EmptyCircuit` — Circuit has no qubits
- `QiskitError::InvalidGate(msg)` — Gate has invalid configuration

## Implementation

Located in `crates/ketgrid-core/src/format/qiskit.rs`:

- `circuit_to_qiskit(circuit)` — Main export function
- `Circuit::to_qiskit()` — Convenience method on Circuit
- `generate_gate_code(gate)` — Maps individual gates to Python
